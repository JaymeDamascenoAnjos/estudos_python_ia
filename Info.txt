** LLM's
Essa é uma das partes mais importantes para entender como as LLMs "pensam". A OpenAI utiliza esses dicionários para criar 
o contexto da conversa.
Imagine que uma conversa com a IA é como uma peça de teatro. Cada dicionário define quem está falando e o que está dizendo:

1. role (O Papel)
    Define a identidade de quem está enviando a mensagem. Existem três papéis principais:

    - system (O Diretor): Aqui você define as regras, o comportamento e os limites da IA. 
        O usuário não vê isso, mas a IA segue como uma "lei".
        Ex: "Você é um analista jurídico", "Responda apenas em JSON", "Seja curto e grosso".

    - user (O Ator/Cliente): É você ou o seu usuário final. 
      É o pedido, a pergunta ou os dados a serem processados.

    - assistant (A IA): São as respostas anteriores da própria IA. Serve para ela "lembrar" o que já foi dito na conversa.

2. content (O Conteúdo)
    É a mensagem de fato. É o texto que o papel (role) está entregando naquele momento.

Exemplo de como a IA "enxerga" a estrutura:    

    messages = [
        # O System dita a regra do jogo
        {"role": "system", "content": "Você é um robô que só fala rimando."},
        
        # O User faz a pergunta
        {"role": "user", "content": "Como está o tempo hoje?"},
        
        # O Assistant (IA) responde seguindo o sistema
        {"role": "assistant", "content": "O sol está brilhando no céu, prepare o seu chapéu!"}
    ]

Por que isso é importante para você?
- Se você colocar uma instrução no user, a IA pode ignorar se o texto for muito longo. Se você colocar no system, 
  a instrução ganha muito mais força.
- Um detalhe técnico: No Python, isso é uma Lista de Dicionários.
- A lista [] mantém a ordem da conversa (o que veio primeiro).
- Os dicionários {} mantêm os detalhes de cada fala.
